# Документация TuringCode: Спецификация высокоуровневого языка для Машины Тьюринга

TuringCode — это специализированный императивный язык программирования, предоставляющий абстракцию высокого уровня над классической одноленточной Машиной Тьюринга (МТ). Данная библиотека реализует полный стек технологий: от лексического анализатора и менеджера памяти до симулятора аппаратного обеспечения.

---

## 1. Архитектура и принцип работы системы

Библиотека функционирует как трехуровневая вычислительная среда:

### 1.1. Аппаратный уровень (Hardware)
Базируется на эмуляторе Машины Тьюринга с бесконечной вправо лентой. Все данные хранятся в **унарной системе счисления** (число $N$ представляется как последовательность из $N$ единиц). 
* **Символ `^`**: Маркер абсолютного начала ленты. Каждая команда гарантированно возвращает каретку к этому маркеру.
* **Символ `#`**: Разделитель регистров памяти.
* **Символ `:`**: Отделяет адрес переменной (символ) от её значения.

### 1.2. Уровень управления памятью (Memory Management)
Реализует **статическое распределение**. Каждая переменная при объявлении получает фиксированный адрес и защитный интервал (**Padding**). 
* **Padding**: Пустые ячейки (`_`), зарезервированные после значения. Если результат операции превышает размер Padding, происходит «переполнение буфера» (Buffer Overflow), и каретка повреждает соседние данные.

### 1.3. Компилятор и ОС (Compiler & OS)
Двухпроходный транслятор преобразует псевдокод в граф состояний МТ.
* **Pass 1**: Сканирование типов, аллокация адресов.
* **Pass 2**: Генерация правил переходов и маршрутизация состояний.
* **Syscalls**: Механизм прерываний для вывода данных (`print`). Симулятор перехватывает специфические состояния, считывает унарный код с ленты и выводит его в десятичном виде.

## 2. Система типов данных

Типизация в TuringCode является строгой. Объявление типа определяет физический размер области памяти на ленте.

| Тип | Буфер (ячеек) | Особенности |
| :--- | :--- | :--- |
| `bool` | 1 | Принимает значения 0 или 1. |
| `byte` | 255 | Оптимален для счетчиков циклов. |
| `char` | 255 | Хранит ASCII-код символа (например, `'A'` = 65). |
| `int` | 10 000 | Стандартный тип для вычислений. |
| `float` | 10 000 | Арифметика с фиксированной точкой (масштаб 100). |
| `int_complex`| 2 x 10 000 | Структурный тип (содержит поля `_r` и `_i`). |

## 3. Особенности написания кода

### 3.1. Объявление переменных
Переменные должны быть объявлены с указанием типа. Допускается объявление без инициализации (по умолчанию 0).
```text
int x = 5
float pi = 3.14
char letter = 'F'
int_complex z
```

### 3.2. Арифметические операции
Поддерживаются бинарные операторы `+`, `-`, `*`, `/`, `%` и унарные `++`, `--`.
* **Деление (`/`)**: Для типа `int` выполняется целочисленное деление. 
* **Float-деление**: Если результат записывается в `float`, компилятор автоматически масштабирует делимое (умножает на 100) для сохранения двух знаков после запятой.
* **Синтаксический сахар**: Поддерживаются операторы `+=`, `-=`, `*=`, `/=`, `%=`.

### 3.3. Управляющие конструкции
* **Цикл `for`**: Требует явного указания всех трех секций.
    ```text
    for ( byte i = 0 ; i < 10 ; i++ ) { ... }
    ```
* **Цикл `while`**: Выполняется, пока значение переменной в скобках больше 0.
* **Ветвление**: Поддерживаются блоки `if`, `else if` и `else`.

### 3.4. Работа с `int_complex`
Тип `int_complex` автоматически разворачивается в две переменные на ленте. При сложении или вычитании двух `int_complex` компилятор генерирует две независимые операции для вещественной и мнимой частей. Доступ к полям осуществляется через суффиксы:
```text
z_r = 10
z_i = 5
```

### 3.5. Вывод данных (`print`)
* **Переменные**: `print x` — выводит значение с учетом типа (для `float` ставит точку, для `char` выводит символ).
* **Строковые литералы**: `print "Text"` — выводит текст напрямую через системный вызов, не используя ресурсы ленты.

## 4. Расширенные механизмы безопасности

### 4.1. NOP-инструкции (No Operation)
Компилятор реализует защиту от разрыва графа состояний. Если в коде встречается инструкция, не требующая действий на ленте (например, только объявление типа), транслятор вставляет «NOP-прыжок». Это гарантирует непрерывность выполнения программы и предотвращает остановку процессора из-за отсутствия правил перехода.

### 4.2. Обработка констант
Числовые литералы (например, `100` или `3.14`) обрабатываются как статические неизменяемые переменные. Им выделяется **нулевой Padding**, что экономит место на ленте и ускоряет поиск, так как эти значения не могут расти в процессе работы.

## 5. Руководство по сборке

Интеграция библиотеки в проект осуществляется через `Makefile`.

1.  **Подготовка**: Исходный код на псевдокоде должен находиться в файле `program.txt`.
2.  **Сборка**:
    ```bash
    make clean
    make
    ```
3.  **Запуск**:
    ```bash
    ./turing_machine
    ```

Система сначала выполнит трансляцию псевдокода в таблицу переходов, затем инициализирует ленту начальными значениями переменных и констант, после чего запустит цикл выполнения шагов Машины Тьюринга до достижения состояния `halt`.
